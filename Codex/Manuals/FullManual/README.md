## Important: Parameter I/O pre-check (Snoop recommended)
When someone asks you to read/write a parameter, **confirm the parameter identity before executing**.

- Use a Snoop tool (Snoop DB / Snoop Current Selection, etc.) or `get_element_info` with `rich=true` to verify:
  - exact parameter (Built-in / Shared / Project)
  - Type vs Instance
  - storage type + units
  - read-only / writable
- If the request is ambiguous (multiple candidates, unclear name, unit/type mismatch), **ask the user to clarify** and do not guess.
- Prefer stable parameter keys in requests: `builtInId` / `guid` (over localized `paramName`).

See also: `Manuals/ConnectionGuide/Revitデータ取得完全ガイド.md` (“パラメータ入出力依頼時の事前確認ルール（Snoop 推奨）”).

## Important: Batch safety (`failureHandling`)
For batch-like commands (heuristic: `elementIds` count >= 5), the router may return `code: FAILURE_HANDLING_CONFIRMATION_REQUIRED` unless you explicitly set `params.failureHandling`. See: [failure_handling](failure_handling.md)
