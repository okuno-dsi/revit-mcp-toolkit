# 柱芯線図ワークフロー（Python Runner）

> 詳細説明版: `Column_Coreline_Workflow_Detailed_JA.md`

> 補足: 元柱ビューなしでの全自動運用は  
> `Column_Coreline_Workflow_FullAuto_JA.md`  
> および `Scripts/PythonRunnerScripts/column_grid_coreline_full_auto.py` を参照。

## 目的
以下の実務フローを、再現しやすい手順として固定化する。

- 通り芯だけのビューを作成
- 柱別ビューの通り芯長さを調整
- 通り芯記号（バブル）を非表示化
- 柱別ビューの右上に構造柱タグを配置（重複なし）
- 柱ビュー（1/100）と通り芯ビュー（1/200）をシートで重ね配置

この文書は **運用手順のみ** を対象とし、DLL差し替え・配布作業は含めない。

## レベル別柱種切替（RC/SRC/S）を前提とした運用ルール
- 同一平面位置でも、レベルにより柱種・外形寸法が異なる場合は、柱ビューをレベル単位で分離する。
1. 柱ビュー作成単位を「平面位置 x レベル」にする。
2. 寸法はレベルごとに再生成する（他レベルの寸法コピー禁止）。
3. シートは原則レベル別（`CGA_<LEVEL>_CORELINE` など）で運用する。
4. 芯ズレ寸法（X/Y）はレベルごとにゼロ判定する（ゼロなら省略）。
5. 将来拡張で柱種プロファイル（RC/SRC/S）を結果JSONに保持する運用を推奨する。

## 事前準備（モデル側）
- 元ビュー（例: `1/2`）は柱と通り芯が表示されている状態にする。
- 必要に応じて次のビューテンプレートを用意する。
  - `通り心だけ`
  - `柱芯線図用ビュー`
- ビュータイプ `柱芯線図` を作成しておく（柱別ビューへ適用するため）。
- シート上のビュータイトルタグ非表示は、運用標準に従い手動で実施する（または既存テンプレートで制御）。

## 使用スクリプト
- `Scripts/PythonRunnerScripts/column_grid_coreline_workflow_sample.py`

## スクリプトが実行する処理
1. 元ビューを複写して通り芯ビュー（`*_GRID_BASE`）を作成  
2. 元ビュー内の構造柱を取得（本数は固定値ではなく自動）  
3. 柱ごとにビューを複写（`*_COL_<elementId>`）  
   - ビュー名プレフィックスは `元レベル名_元ビュー名` を使用
4. 柱外周 +100mm でトリミング  
5. 通り芯を柱中心まわりで `±halfLengthMm` に短縮  
6. 通り芯バブルを非表示  
7. 柱タグを右上へ配置（既存タグは置換）  
8. （有効時）シート1枚に通り芯ビュー + 柱別ビューを重ね配置  
   - 位置合わせは `alignByGridIntersection`（グリッド交点アンカー）

## 主な設定値（スクリプト先頭）
- `SOURCE_VIEW_NAME` : 元ビュー名（空ならアクティブビュー）
- `GRID_TEMPLATE_NAME` : 通り芯ビュー用テンプレート名
- `COLUMN_TEMPLATE_NAME` : 柱ビュー用テンプレート名
- `COLUMN_VIEW_TYPE_NAME` : 柱ビューへ設定するビュータイプ名（既定 `柱芯線図`）
- `COLUMN_MARGIN_MM` : 柱トリミング余白（既定 `100.0`）
- `GRID_HALF_LENGTH_MM` : 柱中心からの通り芯表示半長（既定 `1000.0`）
- `COLUMN_SCALE` : 柱ビュー縮尺（既定 `100`）
- `GRID_SCALE` : 通り芯ビュー縮尺（既定 `200`）
- `PLACE_STRUCTURAL_COLUMN_TAG` : 構造柱タグ配置の有効/無効
- `TAG_OFFSET_RIGHT_MM` / `TAG_OFFSET_UP_MM` : 右上へのタグ配置オフセット
  - 既定: `TAG_OFFSET_RIGHT_MM=100`, `TAG_OFFSET_UP_MM=150`
- `CREATE_SHEET` : シート配置するか
- `MAX_ALLOWED_COLUMNS` : 大量生成防止上限

## 将来拡張の推奨設定（実装時の指針）
- `TARGET_LEVEL_NAMES` : 実行対象レベルの明示配列
- `SHEET_MODE` : `per-level` / `single-sheet`
- `GROUP_BY_PROFILE` : `true` の場合、柱種（RC/SRC/S）で集計・検収
- `VIEW_NAME_FORMAT` : レベル識別が必ず入るテンプレート
- 詳細仕様は `Column_Coreline_Workflow_Detailed_JA.md` を参照

## 実行手順（Python Runner）
1. Revitで対象プロジェクトを開く。  
2. Python Runnerで `column_grid_coreline_workflow_sample.py` を開く。  
3. 先頭設定値を必要に応じて編集。  
4. `Run` 実行。  
5. 出力JSONで次を確認。
   - `columnCount`
   - 各 `items[].ok`
   - `sheet.ok`（シート作成時）

## 期待結果
- 柱本数に応じた `*_COL_*` ビューが作成される。
- 各柱ビューで、通り芯は柱中心周りの短い表示に調整される。
- 通り芯記号（バブル）が非表示になる。
- 各柱ビューで、構造柱タグが右上に1つ配置される（重複なし）。
- 各柱ビューのビュータイプが `柱芯線図` に変更される。
- シート作成時、通り芯ビュー + 柱ビューが重ね配置される。
- source寸法が無い場合も、既定ルール（600/1000/600）で寸法生成が継続される。

## チェックリスト（推奨）
1. 先に 5 本で試験実行し、`ok=true` と `sheet.ok=true` を確認。
2. 問題なければ 30 本へ拡張実行。
3. 各 `*_COL_*` ビューで以下を確認。
   - 柱外周 +100mm でトリミングされている
   - 通り芯が柱中心基準で短縮されている
   - 構造柱タグが右上に1つ表示される（重複なし）
   - 寸法が欠落していない
4. シート側で、通り芯ビューとの重ね位置が崩れていないことを確認。

## トラブルシュート
- `VIEW_TEMPLATE_APPLIED`  
  - テンプレートでロックされている。スクリプト側は `detachViewTemplate=true` を使うが、制約が強いテンプレートでは手動解除が必要。
- 柱数が想定より多い/少ない  
  - 元ビューの表示状態（カテゴリ/フェーズ/フィルタ）を確認。
- シート位置がずれる  
  - グリッド名の整合（同名グリッド）と、元ビューの通り芯形状を確認。
- `requests` が無くて実行できない
  - 依存のない環境でも動くよう、`urllib` フォールバック版スクリプトを使用する。

## 安定運用メモ（2026-02-24）
- 柱ビューへのテンプレート適用は、通り芯調整・寸法処理の後に行う。
- `hide_elements_in_view` は対象IDが空のときスキップする。
- 寸法展開コマンドは、アクティブビュー切替なしで実行する版を使う。
