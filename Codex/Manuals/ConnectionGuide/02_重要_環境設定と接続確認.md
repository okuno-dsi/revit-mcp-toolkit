# 環境設定と接続確認

## 1. 概要
このドキュメントでは、Revit アドインが正しく動作しているかを確認し、標準 MCP 接続が成立しているかを確認する手順を説明します。
新規クライアントの接続確認は、`/mcp` に対する `initialize`、`tools/list`、`tools/call` を基準にしてください。

**用語整理**
- 製品名: `Revit MCP`
- プロトコル名: `Model Context Protocol`
- 既存の `/rpc` と `/job/{id}` は後方互換用のレガシー経路です。

## 2. 前提条件
- Revit が起動しており、対象のプロジェクトファイルが開かれていること。
- Revit MCP アドインがロードされ、サーバーが動作していること。
- PowerShell から HTTP POST を実行できること。

## 3. 推奨: MCP 接続テスト

### 3.1 initialize
```powershell
$base = 'http://127.0.0.1:5210/mcp'
$protocol = '2025-11-05'
$initBody = @{
  jsonrpc = '2.0'
  id = 1
  method = 'initialize'
  params = @{
    protocolVersion = $protocol
    capabilities = @{}
    clientInfo = @{ name = 'manual-check'; version = '1.0' }
  }
} | ConvertTo-Json -Depth 10 -Compress

$init = Invoke-WebRequest -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  Accept = 'application/json, text/event-stream'
  'MCP-Protocol-Version' = $protocol
} -Body $initBody

$sessionId = $init.Headers['MCP-Session-Id']
$init.Content
```

成功条件:
- HTTP 200
- `result.protocolVersion` が返る
- レスポンスヘッダーに `MCP-Session-Id` が含まれる

### 3.2 notifications/initialized
```powershell
Invoke-WebRequest -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  'MCP-Protocol-Version' = $protocol
  'MCP-Session-Id' = $sessionId
} -Body '{"jsonrpc":"2.0","method":"notifications/initialized","params":{}}' | Out-Null
```

### 3.3 tools/list
```powershell
Invoke-RestMethod -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  'MCP-Protocol-Version' = $protocol
  'MCP-Session-Id' = $sessionId
} -Body '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}'
```

成功条件:
- `result.tools` が返る
- `get_open_documents` など既存 Revit コマンドが列挙される

### 3.3a resources/list（互換確認）
```powershell
Invoke-RestMethod -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  'MCP-Protocol-Version' = $protocol
  'MCP-Session-Id' = $sessionId
} -Body '{"jsonrpc":"2.0","id":21,"method":"resources/list","params":{}}'
```

### 3.3b prompts/list（互換確認）
```powershell
Invoke-RestMethod -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  'MCP-Protocol-Version' = $protocol
  'MCP-Session-Id' = $sessionId
} -Body '{"jsonrpc":"2.0","id":22,"method":"prompts/list","params":{}}'
```

### 3.4 tools/call
`get_open_documents` を呼び出し、開いているドキュメント情報を取得します。

```powershell
Invoke-RestMethod -Uri $base -Method POST -ContentType 'application/json' -Headers @{
  'MCP-Protocol-Version' = $protocol
  'MCP-Session-Id' = $sessionId
} -Body '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"get_open_documents","arguments":{}}}'
```

成功条件:
- `result.content` または `result.structuredContent` にドキュメント情報が含まれる
- `isError` が `false`

## 4. レガシー RPC 接続テスト
既存の Python スクリプトや durable queue ベースの運用は継続利用できます。

```bash
python Python/Scripts/Reference/send_revit_command_durable.py --port 5210 --command get_open_documents --output-file C:\path\to\open_documents.json
```

- `result.queued == true` の場合は `jobId` を取り、`/job/{id}` をポーリングしてください。
- 日本語を含む結果を保存する場合は `>` ではなく `--output-file` を使ってください。

## 5. トラブルシューティング
- `ConnectionRefusedError` やタイムアウトエラーが発生した場合は、Revit が起動しているか、アドインが正しくロードされているか、ポート番号が正しいかを確認してください。
- `Missing or invalid MCP session` が返る場合は、`initialize` をやり直して新しい `MCP-Session-Id` を使用してください。
- `Session not initialized` が返る場合は、`notifications/initialized` を先に送ってください。
- `tools/list` に目的のコマンドが出ない場合は、アドイン側の manifest 登録が完了しているか確認してください。
- レガシー `/rpc` 経路で `ok:false` が返る場合は、Revit アドイン側でエラーが発生しています。Revit のダイアログやログを確認してください。
- `/sse`、`/messages`、`/swagger` は現行プロファイルでは提供していません（404）。
- API仕様確認は `/docs/openapi.json` または `/docs/openrpc.json` を利用してください。
